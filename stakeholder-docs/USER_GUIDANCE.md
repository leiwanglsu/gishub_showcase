# User Guidance

## Who This Is For
This guide is for municipal partners, program managers, reviewers, and other non-developer stakeholders who need to understand what GIS Hub can do in the current phase.

## What You Can Use Today
- Preliminary elevation screening support
- Pilot run review and quality-state interpretation
- Stakeholder communication using standardized, governance-aware summaries

## What You Should Not Use It For
- Final FEMA elevation certification decisions
- Legal-grade survey replacement
- Regulatory sign-off without control/checkpoint evidence and licensed review

## Recommended Stakeholder Workflow
1. Confirm objective for the review cycle (screening, planning, pilot demonstration).
2. Review latest annual summary and milestone status.
3. Review readiness statement and risk posture.
4. Confirm that all communications include non-certification language.
5. Capture follow-up actions for next-phase validation.

## Minimum Communication Standard
Use clear, plain language in every briefing:
- outputs are preliminary,
- readiness state is explicit,
- limitations are visible,
- next actions are documented.

## Suggested Audience Packages
- Executive audience: Executive Summary + Annual Report Summary
- Program audience: Milestones + Risk and Governance
- Technical audience: Infrastructure Overview + Pilot Scanning Instructions
